## 1. 개요

### 1.1 과제

국사편찬위원회 색인 XML 사료에서 지명 개체를 자동 추출한다. 측정 단위는 토큰이 아니라 **개체(span)** 이며, 평가는 개체 완전 일치(span exact match) 기준이다. 토큰 micro F1은 O 라벨이 약 98.3%를 차지하는 본 데이터에서 성능을 부풀리므로, 학습·평가 전반에서 개체 단위 P/R/F1을 기준으로 삼는다.

### 1.2 두 모델 구성

같은 작업·같은 아키텍처(SikuRoBERTa + 토큰 분류)로 학습하되, **학습 출처 구성**으로 구분한다.

| 모델 | 학습 출처 | 성격 |
| --- | --- | --- |
| **승정원일기_실록 모델** | 조선왕조실록(전체) + 승정원일기 + 고종·순종실록 및 부록 | 고재현·저정밀 |
| **종합 모델** | 전체 자원(위 + 한국사료총서·고려사·고려사절요·삼국사기·삼국유사) | 저재현·고정밀 |
- *저태깅 5종(한국사료총서·고려사·고려사절요·삼국사기·삼국유사)**은 색인 태깅률이 낮아(시설·국호 점검에서 출처별 태깅률 격차 확인) 발화가 희석되므로, 승정원일기_실록 모델에서 제외한다.

### 1.3 최종 성능 (자체 검증셋, span exact)

| 모델 | val_span_f1 | P | R |
| --- | --- | --- | --- |
| 승정원일기_실록 | 0.9049 | 0.8982 | 0.9118 |
| 종합 | 0.8915 | 0.8803 | 0.9030 |

두 모델의 검증셋은 **구성이 다르다.** 승정원일기_실록 val은 승정원일기+실록만으로 순수 연대기이고, 종합 val은 7종 전체(저태깅 5종 약 5.6% 혼입)이다. 따라서 두 모델의 F1을 직접 비교하면 안 된다 — 저태깅 출처가 정답에 섞이면 모델이 옳게 잡은 표제어도 false positive로 계산되어 F1이 기계적으로 내려간다.

### 2.0 변환 — `0_NER_학습데이터_변환_xml_jsonl.py`

국사편찬위 색인 XML → JSONL 학습데이터.

**입력**: 색인 XML 파일들.
**출력**: `train.jsonl` / `val.jsonl`, `변환_통계.txt`.
**JSONL 필드**: `{'c': 본문, 'l': 문자단위 라벨([0]/[1]/[2]), 'n': 길이, 'id': 기사 ID, 'src': 출처, 'p': 이름 구간 메타필드}`.

형태(라벨은 단일 원소 리스트, BIO 0/1/2):

```json
{"c": "慶尙道晉州府在京南", "l": [[1],[2],[2],[1],[2],[2],[0],[0],[0]], "n": 9, "id": "...", "src": "...", "p": []}
```

핵심 처리:

- 기사 단위 자동 탐지(직속 `<text>`를 가진 levelN), 무표점(한자만 보존), 문자 단위 BIO.
- 기사 ID 해시 분할로 train/val 분리(분할 누수 0 보장).
- 태그 정책: 미등록 태그 기본 '제외+보고'. 포함은 quotation·postScript·proofreading. 제외는 explanation·noteTitle·reference·pTitle·illustration·link·name 등.
- `annotation`은 type별 분기 — 원주만 인라인, 교감주 등은 제외.
- 결락 분절: `missing`·`newChar` 요소 및 원문 □(U+25A1)를 분절 경계로 처리. 결락에 걸친 개체는 라벨 제외(허위 개체 차단).
- 이스케이프 문자 참조(`&#x...;`) 복원으로 확장 영역 한자 회복.
- 비한문 기사 가드: 한글·라틴 실질문자 비율 > 5%면 기사 제외(번역문·영문 차단).
- 출처 식별: sjw(승정원일기), 실록, kr(고려사)/kj(절요)/sg(삼국사기)/sy(삼국유사)/sa(사료총서).
- 관례 점검: 시설·국호 목록의 출처별 태깅률 집계, 출처별 지명 밀도 보고.

### 2.1 검증 — `1_NER_학습데이터_검증.py`

변환 출력 JSONL을 학습 투입 전 독립 검증.

**입력**: `train.jsonl` / `val.jsonl`.
**출력**: `검증_보고.txt`.

검증 항목:

1. 형식 정합: `c`/`l`/`n` 키, `n == len(c) == len(l)`, 라벨 값 {0,1,2}, 한자 외 문자 잔존.
2. BIO 전이 이상: B 없는 I, 시퀀스 첫 글자 I.
3. 분할 누수: 동일 기사 ID가 train·val 양쪽 출현 여부.
4. `p`(이름) 메타필드: 좌표 범위 정합, 지명 라벨과 겹침 집계.
5. 분포: 출처별 청크·개체 수, 라벨 비율, 지명·청크 길이 분포.
6. 육안 검수 표본: 무작위 청크의 라벨 복원 출력.

### 2.2 새니티 — `2_NER_새니티학습_Lightning.py`

본학습 전 단시간 학습으로 붕괴 여부 점검(비용 낭비 방지).

**실행**: `python 2_NER_새니티학습.py --data_dir ~/data --max_steps 300`

점검 항목: ① B-LOC가 0으로 붕괴하지 않는지, ② I-LOC가 B 없이 난발되지 않는지(BIO 위반율), ③ 개체 단위 F1이 학습 진행에 따라 상승하는지.

### 2.3 본학습 — `3_NER_학습_Lightning.py`

**실행**: `python train.py --data_dir data`**재개**: `python train.py --data_dir data --ckpt_path model/checkpoints/last.ckpt`

모델: SikuRoBERTa(`SIKU-BERT/sikuroberta`) + Dropout + Linear(hidden→3). CrossEntropyLoss(ignore_index=-100).

하이퍼파라미터(고정):

| 항목 | 값 |
| --- | --- |
| max_length | 256 (문자 단위) |
| batch_size | 80 |
| accumulate_grad_batches | 2 (유효 배치 160) |
| learning_rate | 2e-5 (AdamW, weight_decay 0.01) |
| 스케줄러 | linear warmup, warmup_ratio 0.1 |
| num_epochs | 3 |
| dropout_rate | 0.1 |
| gradient_clip_val | 1.0 |
| precision | bf16-mixed |
| seed | 42 |
| num_workers | 12 |

학습 운용:

- best/EarlyStopping 모니터: **val_span_f1 (max)**, patience 5. 측정 단위(개체)와 선택 기준의 일치. val_loss는 토큰 평균이라 O 98% 영향이 커 후반 개체 성능과 갈라질 수 있어 쓰지 않는다.
- `val_check_interval=0.25`(에폭당 4회), 전체 val 사용.
- `deterministic=False`(2번과 동일; True는 CUBLAS_WORKSPACE_CONFIG 미설정 시 RuntimeError 위험).
- 데이터셋은 오프셋 인덱싱 + 워커별 파일 핸들 재사용으로 대용량 JSONL 처리.

출력(스튜디오 최상단 기준):

스튜디오 디렉토리 구조 — `train.py`(=3번 스크립트)와 `data/`·`model/`을 같은 층위에 둔다:

```
/teamspace/studios/this_studio/
├── train.py              ← 3_NER_학습_Lightning.py
├── data/
│   ├── train.jsonl
│   └── val.jsonl
└── model/
    ├── checkpoints/      ← best-*.ckpt, last.ckpt
    └── logs/             ← tensorboard/, csv/
```

- `model/checkpoints/best-epoch={E}-step={S}-val_span_f1={F1}.ckpt` (val_span_f1 max 1개)
- `model/checkpoints/last.ckpt` (재개용)
- `model/logs/tensorboard/`, `model/logs/csv/` (train_loss, val_loss, val_span_f1/p/r, val_bio_violations, lr)
- 콘솔: 검증마다 `[VAL]` 요약(개체 F1/P/R, BIO 위반, 토큰별 진단)

평가 지표 정의(2·3·평가 도구 공통):

- `decode_entities`: 엄격 BIO 복원, B 없는 I는 개체 불인정·위반 집계.
- `span_prf`: 개체 완전 일치 TP/FP/FN.

### 2.4 추론·병합 — `4_NER_추론_txt.py`

확정 사양. 단독/병합/검수 병합 세 모드, 단일 파일/폴더 순회 두 입력 방식(전 모드 공통).

병합의 성격: 동일 원문을 두 모델로 각각 추론한 뒤 예측을 후처리로 결합하는 **출력 수준(span-level) 이질 앙상블**이다. 새 기법이 아니라 표준 재현·정밀 결합의 한 형태로 규정한다. 결합이 성립하는 근거는 두 모델의 거동 메커니즘 차이다.

- **승정원일기_실록 모델**: 고태깅 출처만 학습해 확신이 평탄하게 높다 → 고재현·저정밀. 표제어를 폭넓게 잡되 구두점·서술을 넘어 과태깅하는 경향.
- **종합 모델**: 저태깅 5종이 섞여 확신이 문맥에 게이트된다 → 저재현·고정밀. 경계가 깨끗하고 과태깅이 없으나, 단서 없는 표제어를 누락하거나 애매할 때 파편을 태깅.

합집합이 종합의 누락을 메우고, 겹침의 최장 채택이 경계 정밀을 취한다.

**모드**: [1] 단독(ckpt 1개 → `_추론.txt`) / [2] 병합(ckpt 2개, 자동 → `_실록모델` `_종합모델` `_통합본`) / [3] 검수 병합(ckpt 2개, 겹침 사례 수동 선택 → `_검수통합본`).
**입력 방식**: [1] 단일 파일 / [2] 폴더 순회(하위 재귀). 출력물 접미사(`_추론`·`_실록모델`·`_종합모델`·`_통합본`·`_검수통합본`)는 입력에서 제외하여 재추론 방지.

추론 처리:

- 입력에서 한자만 추출하여 모델 투입(학습 분포 일치), `[]` 표시는 원문 좌표로 환원.
- 토크나이즈는 학습과 동일(문자 단위 + word_ids 정렬). max_len·stride는 ckpt에서 자동 결정.
- 추론 span 내부 비한자(구두점·기호·공백)는 경계 오류로 보고 연속 한자 구간별 분할(`split_noncjk`). 예: `金遷。○山` → `金遷` + `山`. 분할 결과 1글자 구간도 유지. — 근거: 지명 내부에 비한자 정답이 없다는 학습 분포 전제에서, 비한자가 낀 구간은 경계 오류다. 여기서 절단(첫 비한자 이후를 버림)과 분할(양쪽 연속 한자 구간을 모두 보존)은 트레이드오프다 — 절단은 비지명 단편을 남기지 않으나 구두점 양쪽이 모두 지명인 경우 한쪽을 잃고, 분할은 그 누락을 막는 대신 비지명 단편(`山`·`西路` 등)을 남긴다. 모델이 지명으로 본 구간을 존중하는 재현 우선으로 분할을 택했고, 잔여 단편은 다운스트림 정제에서 거른다.

병합 규칙(`merge_spans`, 2줄):

1. **합집합**: 두 모델 span을 모두 후보로. — 근거: 위 거동 차이에 따라, 한쪽만 잡은 span을 채택해 종합 모델의 누락을 메운다(재현 확보).
2. **겹침 시 최장**: 서로 겹치는 span끼리 묶어 가장 넓은 것 하나만 남김(동일 길이면 종합 우선). — 근거: 분할(위)이 구두점 흡수를 먼저 제거하므로, 겹침 구간에 남는 경계차는 순수 길이 문제가 된다. 보은·울산 검증에서 경계 상이 9건 중 8건은 넓은 쪽이 정답이었다(정밀 확보).

**검수 병합(모드 3)**: 위 자동 규칙 2(겹침 시 최장)를 **사용자 수동 선택**으로 대체하는 트랙. 겹치지 않는 span은 합집합으로 자동 채택하고, 겹치는 그룹만 콘솔에 제시해 후보를 고르게 한다. 두 모델의 태깅 줄(`실록`/`종합`)을 문맥으로 함께 보여주고, 그룹 내 후보 span을 `[지명] (모델)` 형태로 번호 나열한다 — 1:1 겹침이면 1=실록·2=종합, 드문 다중 겹침(계단식)이면 후보가 더 나열된다. 선택은 번호 하나, 서로 겹치지 않는 번호 복수, 또는 `0`(그 그룹 전부 버림)이 가능하다. 번호 대신 한자를 직접 입력하면 그 자리의 연속 한자 구간 안에서 span을 만들어, 두 모델이 조각내 후보에 없는 개체(`于弗山壇` 등)를 채택한다. 직접 입력이 이미 채택된 좁은 span을 포함하면 좁은 쪽을 제거한다(마지막 선택 우선). 겹침 그룹화는 자동 병합과 동일한 `group_overlaps`를 공유하므로 두 모드의 겹침 판정이 일치한다. 겹침이 없는 줄은 출력 없이 자동 채택한다.

운용:

- 병합은 한 모델로 전 파일을 추론한 뒤 해제하고 다음 모델을 올린다(VRAM 절약, 순차 교체).
- 폴더 모드는 모델을 1회만 올려 전 파일 처리(파일마다 재로딩 방지). 전 파일 결과를 메모리에 보관 후 마지막에 병합·저장.
- 검수 병합(모드 3)도 추론은 모드 2와 동일(모델 1회 로딩). 추론 완료 후 파일별로 겹침 사례를 선택받아 저장하므로, 폴더로 다수 파일을 돌릴 때는 추론 결과가 메모리에 누적된다 — 검수는 소수 파일 단위로 돌리는 것이 적합하다.

ckpt 로더 공통 로직은 `NER_model_loader.py`(bert+classifier 복원).

## 3. 병합 설계 근거 (미학습 데이터 검증)

병합의 핵심 명제는 "고재현 모델과 고정밀 모델의 표준 앙상블이 재현과 정밀을 함께 개선한다"이다. 공인된 정답지가 없으므로 절대 P/R/F1은 평가 목표가 아니며, **단독 모델 대비 상대 개선**을 사례로 제시한다. 고전 사료에서는 "무엇을 지명으로 볼 것인가"가 색인 관례에 따라 흔들리므로(태깅률 출처별 격차), 단일 벤치 수치는 그 관례성을 가린다.

### 3.1 모델 예측 (신증동국여지승람_보은현, 대동지지_울산 사례)

- **종합 단독**: 지리지 산천·시설 표제어를 대량 누락(보은 41 span, 울산 140 span — 고유 기준).
- **승정원일기_실록 단독**: 같은 텍스트에서 표제어를 폭넓게 포착(보은 107, 울산 270), 단 구두점 흡수·파편 발생.
- **통합본**: 합집합으로 누락 복구 + 겹침 최장으로 경계 정리(보은 115, 울산 274).

### 3.2 겹침 최장 규칙 채점 (경계 상이 쌍)

보은 2건 + 울산 7건, 총 9건 중 8건에서 "넓은 쪽 채택"이 정답:

- 정답 사례: `于弗山`(종합 `弗山`), `處容巖`(종합 `巖`), `鶴城`(종합 `城`), `開雲浦鎭`(종합 `開雲浦`), `于弗山壇`(종합 `弗`), `熊村`(실록 `熊`), `林堰`(종합 `堰`), `屛風淵`(종합 `風淵`).
- 유일한 반례: `隷良州` — 종합이 동사 `隷`를 흡수, 넓은 쪽 채택 규칙이 이를 교정하지 못한다. 다만 이는 종합 모델의 단독 오답이 전파된 것이며, 규칙 신설로 발생한 오류가 아니다.

검수 비용 측면에서도 최장 채택이 유리하다. 경계가 좁게 잡힌 오류는 검수자가 누락분을 판단해 경계를 확장해야 하는 반면, 넓게 잡힌 오류는 과포함된 글자를 제거하면 된다. 최장 채택은 겹침 구간의 경계 오류를 후자 형태(과포함)로 집중시켜, 검수가 확장이 아닌 제거 작업으로 단순해진다.

### 3.3 잔여 한계

- 분할 잔여 단편(`路`·`內`·`西` 등 비지명 1글자)이 통합본에 남는다.
- `隷` 류 선행 허사 흡수.

이들은 모델·규칙의 결함이라기보다, 지명 경계가 본래 확정적이지 않은 자료 특성이 표면화된 지점이다. 추론·병합 단계에서 풀지 않고 다운스트림에서 거른다.

## 4. 부속 도구

### 4.1 모델 평가 — `NER_모델평가.py`

정규 파이프라인 외 횡단 유틸리티. 채부 판정·도메인 격차·전후 비교에 반복 사용.

**실행**: `python test.py --ckpt {ckpt} --data data/val.jsonl --out 모델평가.txt`
사전 분석 포함: `--gazetteer {csv}`. 표본 실행: `--limit 5000`.

측정: ① span exact TP/FP/FN(3번 검증 로직과 동일, 수치 교차 확인 가능), ② 경계 불일치(겹치되 exact 아닌 예측), ③ FN 분석(표층형 빈도·길이·출처별, 사전 수록 여부, 사전 수록 FN의 모델 지명 확률 분포), ④ FP 분석, ⑤ 사전 최장일치 구간 확률 임계값 0.1~0.5 시뮬레이션. ④·⑤는 `--gazetteer` 지정 시에만 의미.

### 4.2 지명DB 정제 — `지명DB_정제.py`

지명 사전을 반복 정제. 출처 통합으로 갱신되므로 매 정제마다 감사 가능한 보고 생성.

정제 규칙: ① 쉼표(,·、) 병기 분리, ② 결손·생략 표지(`?■○-ㅡ`) 포함 부분 제외, ③ 각 부분은 선두 한자 연쇄만 취함(허위 항목 생성 차단), ④ 2자 미만 제외(1자 지명은 사전 매칭 대상 아님), ⑤ 거리·방위 파편 제외(방위·숫자·里로만 구성 + 행정단위(郡縣府州)+방위 2자; 집합 밖 글자가 있으면 통과해 江東·南山·東面 등 보존), ⑥ 중복 제거·정렬·utf-8-sig 저장.

**출력**: 정제본 CSV(헤더 name), 정제_통계.txt.

### 4.3 지명DB 추론 검수 — `지명DB_추론_검수.py`

[] 태깅된 추론 출력에서 정제 지명DB 표제어를 본문 최장일치로 찾아 모델 태깅과 함께 검토한다. DB 매칭은 학습에서 배제한 방식이며 여기서는 검수 보조로만 되살린다(동형이의 위험 때문에 자동 삽입이 아니라 사람이 선택).

화면 표시는 모델 `[]`, DB `<>`. 분기: ① 모델 단독·완전 일치 → 자동 유지, ② DB가 모델에 포함(모델이 더 김) → 모델 최장 자동 채택, ③ DB 단독·경계 교차 → 선택지 제시. 선택지에서 번호 외에 한자를 직접 입력하면 그 자리의 연속 한자 구간 안에서 span을 만들어, 조각난 후보로 못 만드는 개체(屹紅溫井·楊山古城 등)를 채택한다. 직접 입력이 기존 채택을 포함하면 좁은 쪽을 제거(마지막 선택 우선). 출력은 `입력_DB검수.txt`.

## 6. 재현 절차 요약

1. **변환**: 색인 XML → `0_NER_…py` → train/val.jsonl (+ 변환_통계).
2. **검증**: `1_NER_…py`로 형식·BIO·누수·분포 확인 (검증_보고).
3. **새니티**: `2_NER_…py --max_steps 300`으로 붕괴 점검.
4. **본학습**: `3_NER_…py --data_dir data` (val_span_f1 max, 3에폭, bf16). best ckpt 산출.
    - 두 모델은 각각 별도 데이터(승정원일기_실록 / 종합)로 동일 절차 반복.
5. **추론·병합**: `4_NER_추론_txt.py` 모드 2 + 입력 방식 선택, 두 ckpt 지정 → 통합본. 겹침 사례를 직접 판단하려면 모드 3(검수 병합) → 검수통합본.
6. **검수**: 통합본을 사례 기반으로 검토. 잔여 단편·선행 허사는 다운스트림 정제 대상.
